# 자료구조 (Data Structure)

### 자료구조의 의미

- 컴퓨터에 데이터를 저장하고 정리하는 방식

### 왜 자료구조를 배워야 할까?

- 적절한 자료구조 방식 채택 → 코딩 용이 → 신속 정확한 테스트 가능 → 유지/보수 용이
- 알고리즘 = 자료구조 + 문제 해결 절차

### 종류

- 선형 구조 : 데이터가 순차적으로 연결된 형태
    - 배열, 연결 리스트, 스택, 큐

- 비선형 구조 : 데이터가 계층적으로 연결된 형태
    - 트리, 그래프

### 시간 복잡도

- 알고리즘이 실행되는 데 걸리는 시간을 분석하여 효율성을 평가하는 척도

### 공간 복잡도

- 알고리즘이 실행되는 동안 사용되는 메모리 공간의 양을 분석하여 효율성을 평가하는 척도

### Big-O 표기법

<img alt="빅오표기법.png" src="/김재혁/251229/image/빅오표기법.png">

- 프로그램의 성능을 정확히 파악하는 것이 불가능

  → 대략적인 성능 비교를 위해 **상대적 표기법**을 사용하기로 함

- 알고리즘의 효율성은 데이터 입력값(n)의 크기에 따라 영향을 미침.
- 기본적으로 데이터 입력값(n)이 충분히 크다고 가정을 하기 때문에 아래 두 가지를 기억해야 함.
    1. **상수항 무시**
    2. **가장 큰 항 외에 무시**

# 해시 테이블 (Hash Table)

- 다른 유튜버의 영상을 다운받아 내 유튜브에 그대로 업로드하면?

  → “중복된 영상”으로 분류하여 정상 업로드 X

- 해시 (Hash) : 잘게 자르다, 마구 섞다
- 키 값을 분해하고 혼합하여 인덱스로 만듦

<img alt="해시테이블.png" src="/김재혁/251229/image/해시테이블.png">

### 구조

- 키 (Key)
- 해시 함수 (Hash Function)
- 해시 값 (Hash Value)
- 해시 테이블 (Hash Table)

### 기본 원리

1. Key를 입력
2. 해시 함수를 통해 해시 값으로 변환
3. 해시 값을 통해 해시 테이블 접근
4. 해시 테이블의 값 출력

### 해시 충돌 (Collision)

- 해시 함수가 **다른 키**에 대해 **같은 해시 값**을 반환

  → 테이블의 같은 위치에 두 개 이상의 데이터가 저장되려는 현상


### 분리 연결법 (Chaining)

- 해시 테이블 저장 공간 이외에 추가 메모리를 활용
    - 장점 : 구현 쉬움, 충돌이 많아도 데이터 손실 X → 안정적
    - 단점 : 최악의 경우 시간 복잡도 O(n)

### 개방 주소법 (Open Addressing)

- 해시 테이블 저장 공간 내에서 충돌 문제 해결
    1. 선형 조사 (Linear Probing)

       해당 Index로부터 고정 폭 만큼 이동하며 발견된 빈 버킷에 데이터 저장

        - 장점 : 구현이 쉬움
        - 단점 : 1차 군집화(Clustering)가 심함
    2. 이차 조사 (Quadratic Probing)

       해시의 저장 순서 폭을 제곱으로 저장 ( 1^2, 2^2, 3^2, … )

        - 장점 : 군집화 완화
        - 단점 : 테이블 크기 설계에 신경 써야함
    3. 이중 해싱 (Double Hashing)

       해시된 값을 한 번 더 해싱하여 규칙성 제거

        - 장점 : 충돌 해결 성능이 좋음
        - 단점 : 구현이 가장 복잡함
- 예시 이미지

  <img alt="개방주소법.png" src="/김재혁/251229/image/개방주소법.png">


### 특징

- 장점
    - 데이터 저장 및 검색 속도가 빠름 (성능 우수)
    - 충돌이 없을 경우 시간 복잡도는 O(1)
    - 중복 확인이 쉬움
- 단점
    - 저장 공간이 많이 필요함
    - 최악의 경우 시간 복잡도는 O(n)
    - 별도의 자료 구조가 필요할 수 있음
- 주요 용도
    - 데이터의 저장, 삭제, 검색이 많이 필요한 경우
    - 웹 페이지에서 캐시를 구현하는 경우
