# 🌳 Tree (트리) 자료구조 정리

## 1. 트리 개념
트리는 **부모–자식 관계로 이루어진 계층적 자료구조**이다.  
하나의 루트(Root)에서 시작해 여러 하위 노드로 확장된다.

---

## 2. 트리 vs 그래프

| 구분 | 트리 | 그래프 |
|---|---|---|
| 루트(Root) | O | X |
| 사이클(Cycle) | X | O |
| 경로(Path) | 유일 | 여러 개 가능 |
| 구조 | 계층적 | 일반적 |

---

## 3. 트리 구성 요소

| 용어 | 설명 |
|---|---|
| Node | 데이터를 저장하는 기본 단위 |
| Root | 트리의 시작 노드 |
| Parent | 상위 노드 |
| Child | 하위 노드 |
| Leaf | 자식이 없는 노드 |
| Edge | 노드를 연결하는 선 |
| Depth | 루트에서 특정 노드까지의 거리 |
| Height | 해당 노드에서 가장 깊은 노드까지의 거리 |
| Subtree | 어떤 노드를 루트로 하는 부분 트리 |

---

## 4. 트리 종류

### 4.1 일반 트리 (General Tree)
- 자식 노드 개수에 제한이 없는 트리
- 노드마다 자식 수가 다를 수 있음
- 이진 트리처럼 left / right 구분 없음

---

### 4.2 이진 트리 (Binary Tree)
- 각 노드가 **최대 2개의 자식 노드**를 가질 수 있음
- 보통 `left`, `right`로 구분
- 값의 크기 규칙은 없음
- 트리 순회 개념이 가장 명확하게 정의됨

---

### 4.3 이진 탐색 트리 (Binary Search Tree, BST)
- 이진 트리에 **정렬 규칙**이 추가된 구조
- 규칙  
  - 왼쪽 서브 트리 < 부모 노드  
  - 오른쪽 서브 트리 > 부모 노드

---

### 4.4 완전 이진 트리 (Complete Binary Tree)
- 마지막 레벨을 제외한 모든 레벨이 **꽉 차 있는** 이진 트리
- 마지막 레벨은 왼쪽부터 채워짐

---

### 4.5 힙 (Heap)
- **완전 이진 트리 기반** 자료구조
- 최대 힙(Max Heap): 부모 ≥ 자식  
- 최소 힙(Min Heap): 부모 ≤ 자식
- 우선순위 큐 구현에 사용됨

---

### 4.6 트라이 (Trie)
- 문자열 저장과 탐색에 특화된 트리
- 접두사(prefix) 기반 검색에 효율적
- 자동 완성, 사전, 검색 엔진 등에 활용

---

## 5. 트리 순회 (Traversal)

### 5.1 깊이 우선 탐색 (DFS)

- **전위 순회 (Preorder)**  
Root → Left → Right


- **중위 순회 (Inorder)**  
Left → Root → Right


- **후위 순회 (Postorder)**  
Left → Right → Root

---

### 5.2 너비 우선 탐색 (BFS)
- 레벨(level) 순서대로 탐색
- 주로 Queue 자료구조 사용

---

## 6. 시간 복잡도

| 구조 | 탐색 시간 복잡도 |
|---|---|
| 이진 트리 | O(N) |
| 이진 탐색 트리 (평균) | O(log N) |
| 이진 탐색 트리 (최악) | O(N) |

---

## 7. 트리 문제 접근 순서

1. 이진 트리 구조인가?
2. 값의 정렬 규칙이 있는가? → **BST**
3. 최대/최소 반복 추출인가? → **Heap**
4. 문자열 / 접두사 문제인가? → **Trie**
5. 순회 결과를 묻는가? → **DFS / BFS**
